**User**

莆田一个剪纸合作社每月接300多份定制订单，客户要求各不相同——有人要婚庆双喜配龙凤、有人要企业LOGO剪影——合作社秘书靠Excel登记需求，经常漏单或把甲的要求发给乙。请搭建一个在线商城，客户能浏览合作社的作品集、选择纹样模板下单定制（填写尺寸、配色、祝福语等参数），也能上传参考图要求原创设计。后台管理订单全流程：接单→分配匠人→制作中→质检→发货，状态变更时自动给客户发站内通知。匠人端能看到分配给自己的任务和截止日期，上传制作过程照片供客户查看。支付对接模拟接口即可，但下单时库存和匠人排期要做乐观锁防止超卖——同一个匠人同一时段被分配的任务不能超过其产能上限。技术栈前端用Vue 3，后端FastAPI，PostgreSQL存业务数据，Redis做库存缓存和会话管理，用Celery处理订单状态变更的异步通知任务。前端交付商城首页、作品详情页、定制下单表单、匠人工作台、订单跟踪页；后端交付用户认证、商品与订单CRUD、匠人分配与排期、库存乐观锁控制、通知推送等接口，附API接口文档和数据库ER图，以及docker-compose一键启动配置。

## 关键设计

**乐观锁实现：**
- 库存扣减：`UPDATE products SET stock=stock-1, version=version+1 WHERE id=? AND version=? AND stock>0`，影响行数为0则抛出"库存不足"异常
- 排期分配：`UPDATE artisan_schedules SET assigned_count=assigned_count+1, version=version+1 WHERE artisan_id=? AND year_month=? AND version=? AND assigned_count<capacity`，影响行数为0则抛出"匠人当月产能已满"异常

**状态变更通知：** 订单状态更新时通过 `send_order_status_notification.delay()` 异步调用Celery任务，创建站内通知

**权限控制：** 基于 JWT token 中的 role 字段，通过 `get_current_user` / `require_admin` / `require_artisan_or_admin` 三个依赖项实现分级权限

### 🔒 乐观锁机制（核心亮点）

**库存乐观锁** — 下单时防止超卖：
```sql
UPDATE products SET stock=stock-1, version=version+1
WHERE id=? AND version=? AND stock>0
```

**匠人排期乐观锁** — 防止同一匠人同月超产能：
```sql
UPDATE artisan_schedules SET assigned_count=assigned_count+1, version=version+1
WHERE artisan_id=? AND year_month=? AND version=? AND assigned_count<capacity
```

影响行数为0时自动抛出异常，前端提示"库存不足"或"匠人当月产能已满"。
